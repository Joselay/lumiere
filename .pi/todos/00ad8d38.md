## Why
Current backtesting fetches only one OKX candle page (`HistoricalCandleRequest.limit`, default 300). That is only a few hours on `1m`, too little to prove expected edge. A quick current default run showed BTC/ETH MA crossover losing after modeled costs while buy-and-hold was positive.

## Scope
- Add paginated OKX historical candle fetching over configurable date ranges.
- Store reproducible datasets locally (CSV/SQLite/Parquet) with symbol, bar, fetch time, and checksum/version metadata.
- Add CLI flags for `--start`, `--end`, `--bar`, and cached/offline backtests.
- Add chronological train/validation/test and rolling walk-forward reports.
- Include no-trade and buy-and-hold baselines for every split.

## Acceptance criteria
- Can backtest BTC-USDT and ETH-USDT over months of data without lookahead.
- Report clearly shows in-sample vs out-of-sample net PnL, drawdown, profit factor, Sharpe/Sortino, trade count, and baseline comparisons.
- Tests cover pagination ordering/deduplication and split boundaries.
