## Why
Several high-value risk controls are disabled by default: performance gate, drawdown limit, spread guard, expected-edge buffer, and max-risk-per-trade. Settings named `*_PCT` are fractions in code (`0.01` means 1%), which can be misconfigured as `1` = 100%.

## Scope
- Rename/document risk fraction settings or parse human percentages safely.
- Add validation warnings/errors for dangerous demo/live configs: open Telegram chat access, no performance gate, no spread/cost gate, no drawdown cap, oversized exposure.
- Compute persistent account/equity high-water mark so live max drawdown gate actually trips.
- Fetch order-book cost estimates whenever a spread/cost/edge guard is configured, not only when max spread is set.
- Provide a `lumiere-risk-audit` command that prints pass/fail readiness for current env.

## Ready when
A misconfigured bot fails fast before trading instead of silently running with disabled profitability/safety gates.
