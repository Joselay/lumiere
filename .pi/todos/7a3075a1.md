## Why
Backtests support stop-loss, take-profit, trailing-stop, and max-bars exits, but the live engine only follows strategy buy/sell signals. Existing demo positions can cause repeated small sells and stale exposure without a live exit policy tied to entry state.

## Scope
- Persist per-instrument entry price/time/size and active strategy reason.
- Apply live stop-loss, take-profit, trailing-stop, max-bars/max-age, and max-unrealized-loss exits before normal strategy decisions.
- Detect unexpected starting inventory and require an explicit flatten/ignore/adopt decision before trading it.
- Refresh or update account snapshot after each order/fill so multi-symbol decisions use current exposure and cash.
- Add tests for stale positions, partial fills, panic/flatten, and exit priority.

## Ready when
The bot can explain every open position and has deterministic exit rules even if the original signal disappears.

## Completed
- Added persisted live position state/provenance with position explanations.
- Added block/adopt/flatten/ignore handling for unexpected starting inventory.
- Added live protective exits for stop-loss, take-profit, trailing stop, max bars, max age, and max unrealized loss before normal strategy decisions.
- Refreshed account snapshots after order/fill reconciliation so later symbol decisions see current exposure.
- Added tests for unexpected inventory blocking/flattening, exit priority, stale max-age exits, and partial-fill state reconciliation.
