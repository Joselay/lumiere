## Why
`RiskConfig.performance_gate_required` exists, but `OKXDemoClient.fetch_account_snapshot()` does not currently compute a real rolling gate; enabling it would block orders because `performance_gate_passed` defaults false. The engine needs forward evidence before placing orders.

## Scope
- Add a persistent paper-trading/shadow ledger that records strategy decisions, simulated fills, modeled fees/spread/slippage, and rolling PnL.
- Compute `performance_gate_passed` from recent paper/live metrics, not a static default.
- Block real/demo orders unless the active strategy passes rolling gates: min trades, positive net PnL after costs, acceptable drawdown, profit factor, and slippage.
- Add Telegram commands/report fields explaining gate status and rejection reason.

## Acceptance criteria
- With `RISK_REQUIRE_PERFORMANCE_GATE=true`, orders pass only after rolling evidence passes the gate.
- Gate state survives restarts.
- Tests cover blocked, passing, and decayed-gate scenarios.
