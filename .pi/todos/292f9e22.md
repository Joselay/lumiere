## Why
The unit suite passes, but coverage is weakest in live paths: `okx_client`, `telegram_bot`, `telegram_ui`, `engine.run_forever`, and the CLIs. These are the exact paths that can lose money or hide bad evidence.

## Scope
- Build a deterministic fake OKX exchange adapter with candles, order book, balances, fills, partial fills, rejects, and API failures.
- Run engine ticks end-to-end through strategy -> risk -> order -> fill -> attribution -> Telegram text.
- Add CLI smoke tests for backtest/optimizer/evidence using offline cached fixtures.
- Add regression tests for previous logging crash behavior and structured exception paths.
- Set a coverage target for money-path modules rather than total-line vanity coverage.

## Ready when
A realistic fake-exchange scenario catches broken PnL, fill attribution, risk gates, and notifications before demo trading does.
