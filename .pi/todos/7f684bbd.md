## Why
Backtests currently model strategy fills and costs, but do not run the same live `RiskManager` rules for cooldown, spread/cost gates, position clamps, daily limits, drawdown de-risking, or post-clamp minimum size.

## Scope
- Introduce a shared simulation/execution module used by backtest and paper trading.
- Apply the same risk assessment, size clamp, cooldown, min order, max position, max exposure, and cost gates in backtest/paper/live.
- Reject or hold after clamp if the clamped order falls below min size instead of letting live order validation become an engine failure.
- Make optimizer candidate reports include risk rejection counts and opportunity cost from blocked signals.
- Test parity: a fake exchange scenario should produce matching decisions/fills across backtest, paper, and live simulator.

## Ready when
A strategy cannot pass research because of assumptions that the live engine would reject or size differently.

## Completion evidence
- Added `lumiere.execution.prepare_execution_plan` shared by live engine, backtester, and paper ledger to apply RiskManager assessment, clamps, and post-clamp minimum-size rejection.
- Backtest and paper snapshots now carry cost/spread/drawdown/trade-count state into risk checks; backtest/paper configs accept `RiskConfig`.
- Live engine uses the shared execution plan and blocks `clamped_order_size_below_minimum` without order submission/failure.
- Backtest reports include `risk_rejection_count`, per-reason `risk_rejections`, and `blocked_signal_opportunity_cost_usdt`; optimizer rank/report payloads include these fields through backtest reports.
- Added fake-exchange parity tests covering matching backtest/paper/live fill size, post-clamp live block, and backtest/paper risk rejection evidence.
- Verification: `uv run ruff check .`; `uv run pytest -q`.
