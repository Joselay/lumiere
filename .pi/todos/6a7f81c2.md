## Why
Profit can never be guaranteed, but we can require strong evidence before risking more size. The project should have an explicit promotion process from backtest -> paper -> small demo -> larger demo.

## Scope
- Define go/no-go thresholds for each stage: out-of-sample PnL, profit factor, max drawdown, Sharpe/Sortino, min trades, slippage, and baseline outperformance.
- Require a minimum forward paper/demo observation period before increasing size.
- Document rollback rules: pause on drawdown, gate failure, abnormal costs, API errors, or market regime mismatch.
- Add commands/scripts to produce the evidence packet for each promotion decision.

## Acceptance criteria
- A human-readable runbook lives in the repo and is linked from README.
- Bot config defaults stay conservative until the runbook gates are met.
- CI/test fixtures verify evidence reports can be generated without live credentials.
