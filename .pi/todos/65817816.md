## Why
Backtests decide using a candle's close and fill on that same close. Live `get_candlesticks` parsing also ignores OKX's confirm flag, so the bot may act on a still-forming candle that can repaint.

## Scope
- Parse and preserve OKX candle confirmation status.
- Live engine should trade only on closed/confirmed candles, or explicitly mark intrabar mode as experimental.
- Backtester should support next-bar execution (signal on closed candle N, fill at open/VWAP of N+1) and compare with current same-close mode.
- Model stop-loss/take-profit/trailing-stop with pessimistic intrabar sequencing using high/low, not only close.
- Add regression tests proving split/walk-forward reports do not leak future candles.

## Ready when
Research reports and live decisions use the same no-lookahead timing assumptions, and the report states execution timing explicitly.

## Completion evidence
- `MarketCandle.confirmed` is parsed from OKX confirm flags, persisted in historical CSVs, and unconfirmed live candles are filtered by `OKXDemoClient.fetch_candles`.
- `TradingEngine` filters to confirmed candles by default; `EngineConfig.experimental_intrabar=True` is the explicit opt-in to intrabar/unconfirmed decisions.
- `Backtester` defaults to `execution_timing="next_open"`, ignores unconfirmed candles, includes same-close comparison metrics, and reports timing assumptions.
- Stop-loss/take-profit/trailing-stop exits use high/low with pessimistic sequencing.
- Regression tests cover unconfirmed candle filtering, next-open fills, intrabar exits, and split/walk-forward no-lookahead report timing.
- Verification: `uv run ruff check .`; `uv run pytest -q`.
